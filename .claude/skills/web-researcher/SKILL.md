# Web Researcher — 联网搜索学习资源

> 为学习过程注入互联网的最新知识：文档、讨论、开源项目、论文。

---

## 核心定位

静态教程有滞后性。本 Skill 利用联网搜索能力，获取最新的文档、
社区最佳实践、相关开源项目和学术论文。也是 AI 自主探索学习方向的入口。

---

## 触发条件

- 需要查找官方文档
- 需要了解社区最佳实践
- 需要搜索相关开源项目
- 需要查找论文或 RFC
- 需要对比不同技术方案的最新状态
- AI 自主建议学习方向时

---

## 搜索类别

> **搜索技巧和评估方法详见 `docs/search-strategies.md`**，包含：搜索优先级、限定站点技巧、时间过滤策略、结果质量评估标准。以下为速查分类表。

| 类别 | 搜索目标 | 典型查询 |
|------|---------|---------|
| 官方文档 | 最新 API、迁移指南、changelog | `site:docs.rs <topic>` |
| 社区讨论 | StackOverflow、Reddit、HackerNews | `site:reddit.com/r/rust <topic>` |
| 开源项目 | GitHub 优秀项目、awesome 列表 | `awesome <topic> github` |
| 博客文章 | 深入教程、经验分享、性能测试 | `<topic> deep dive blog` |
| 论文/RFC | 学术论文、技术规范 | `<topic> RFC` `<topic> paper` |
| 对比评测 | benchmark、技术选型 | `<techA> vs <techB> benchmark 2026` |

## 输出结构

每次搜索输出结构化报告到对应学习计划或 `docs/deep-dives/`：

```markdown
# 搜索结果: [搜索主题]

> 搜索日期: [日期]
> 搜索意图: [为什么要搜索这个]

## 官方资源
- [官方文档 — 最新版本](url)
- [迁移指南](url)
- [API 参考](url)

## 社区资源
- [文章标题](url) — [一句话描述]
- [讨论帖](url) — [关键结论]

## 开源项目
- [项目名](url) ⭐ stars — [用途和特点]

## 推荐阅读顺序
1. [先读这个 — 原因]
2. [再读这个 — 原因]
3. [深入 — 原因]

## 关键发现
[从搜索结果中提炼的 3-5 个最重要的知识点或趋势]
```

---

## 工作流

1. **明确搜索意图** — 确定要解决什么问题
2. **多类别搜索** — 至少搜索 3 个类别
3. **过滤和排序** — 按相关性和权威性排序结果
4. **提炼关键发现** — 不罗列链接，总结关键信息
5. **整合到学习计划** — 将高质量资源加入 resources.md

---

## 护栏规则

1. **标注日期** — 所有搜索结果标注获取日期，避免信息过期
2. **标明权威性** — 官方文档 > 知名博客 > 社区帖子
3. **提炼不罗列** — 输出必须有关键发现，不能只是链接列表
4. **不搜索盗版** — 不搜索和推荐盗版书籍/课程
5. **推荐阅读顺序** — 帮用户做信息过滤，降低选择成本
