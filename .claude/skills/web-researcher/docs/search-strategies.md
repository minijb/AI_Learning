# 搜索策略指南

## 搜索优先级

1. **官方文档优先** — 永远先查官方 docs、spec、RFC
2. **Awesome 列表** — 快速了解生态全貌
3. **深度文章** — 找"deep dive"、"under the hood"、"how X works"
4. **性能数据** — 找 benchmark、性能对比文章
5. **社区问答** — StackOverflow 解决具体问题

## 搜索技巧

### 限定站点
- `site:docs.rs` — Rust 官方文档
- `site:github.com` — GitHub 仓库
- `site:reddit.com/r/rust` — Reddit 社区
- `site:stackoverflow.com` — StackOverflow

### 限定时间
- 新技术（<1 年）: 搜索最近 3 个月
- 成熟技术: 搜索最近 1 年
- 基础概念: 不限时间

### 关键词组合
- `[技术名] best practices 2026`
- `[技术名] vs [对比技术] comparison`
- `[技术名] production experience`
- `[技术名] common pitfalls`
- `[技术名] performance benchmark`

## 结果评估

评估每个搜索结果:
- **时效性**: 发布时间是否仍然有效
- **权威性**: 作者/组织是否可信
- **深度**: 是入门教程还是深入分析
- **实操性**: 包含可运行代码还是纯理论
