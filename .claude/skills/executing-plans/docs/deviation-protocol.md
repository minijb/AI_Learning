# Deviation Protocol — 偏离处理协议

> 当执行者想偏离计划时，强制执行此协议。

---

## 核心原则

> **计划是唯一真实来源。偏离 = 执行者和制定者各自有不同版本的"真理"。**

---

## 触发场景与处理

| 触发场景 | 标记 | 处理 |
|---------|------|------|
| 计划步骤有歧义 | `[NEEDS_CLARIFICATION]` | 停 + 列出歧义点让用户选择 |
| 发现计划遗漏了边界情况 | `[PLAN_GAP]` | 停 + 说明遗漏 + 建议补充 |
| 有"更好"的实现思路 | `[IMPROVEMENT_IDEA]` | 停 + 说明你的想法 + 让用户决定 |
| 验证失败 | `[VERIFY_FAIL]` | 停 + 报告实际输出 vs 预期 |
| 文件路径不存在 | `[PATH_MISMATCH]` | 停 + 报告路径 + 不猜测正确路径 |
| 依赖缺失 | `[MISSING_DEP]` | 停 + 报告缺失的依赖 |

---

## 处理流程

```
发现偏离 → 标记类型 → 写清楚问题 → 停 → 等待用户回复
                                          ↓
                              用户确认 → 按确认方向继续
                              用户拒绝 → 严格按原计划继续
                              用户更新计划 → 重新加载计划
```

---

## 反模式（禁止行为）

| ❌ 反模式 | 后果 |
|----------|------|
| "我猜应该是 X" | 猜错返工 |
| "验证失败但应该没关系" | 累积未验证步骤 → 后期灾难 |
| "我顺便优化一下" | Scope creep |
| "路径不对，我搜一下正确路径" | 可能搜到错误路径 |
| "虽然没在计划里，但显然需要 X" | 给执行者错觉——以为自己比计划懂得多 |
