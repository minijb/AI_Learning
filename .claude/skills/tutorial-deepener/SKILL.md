# Tutorial Deepener — 教程深化与扩展

> 拿到一个已有教程，找出它的知识盲区，补全它。

---

## 核心定位

用户已经有了一份教程（可能来自网络、书籍、文档），但觉得不够深入或不够全面。
本 Skill 负责在不改变原教程核心结构的前提下，做"填空"和"加深"。

---

## 触发条件

- 用户提供了教程链接 + "深化一下"
- 用户粘贴了教程文本 + "帮我扩展"
- 用户说"这个教程太浅了，补充一下原理"
- 任何需要对已有教程做增强的请求

---

## 深化维度

> **深化策略和方法论详见 `docs/deepening-strategies.md`**，包含：深化优先级排序、各维度深化程度控制、浅/深层教程判断标准。以下为速查维度列表。

对教程的每个知识点，检查以下维度是否缺失：

| 维度 | 说明 | 示例问题 |
|------|------|---------|
| 前置知识 | 教程假设读者已懂但没解释的概念 | "教程直接用了闭包但没解释什么是闭包" |
| 原理推导 | 只讲了"怎么做"没讲"为什么这么做" | "为什么用 B+ 树而不是红黑树？" |
| 边界情况 | 未讨论的异常输入、极限值、并发场景 | "空数组传入会怎样？" |
| 性能特征 | 时间/空间复杂度，实际 benchmark | "这个操作在 10 万条数据下会慢吗？" |
| 对比方案 | 同类技术的替代方案和选型原因 | "为什么选 Redis 而不是 Memcached？" |
| 实战陷阱 | 生产环境中容易踩的坑 | "这个模式在大流量下会导致连接池耗尽" |
| 演进历史 | 技术为什么被发明，解决了什么问题 | "React 的 Virtual DOM 解决什么痛点？" |

---

## 输出结构

深化后的教程输出到 `docs/deep-dives/` 或合并到已有学习计划中：

```markdown
# [原教程标题] — 深化版

## 原教程摘要
[简要总结原教程覆盖的范围]

## 深化内容

### [知识点 1] 的前置知识补充
[原文没有解释但需要的前置概念]

### [知识点 1] 的原理推导
[从第一性原理出发的解释]

### [知识点 2] 的边界情况分析
[异常输入、并发、极限值等边缘场景]

### [知识点 2] 的对比方案
[同类技术对比表格 + 选型建议]

## 实战案例
[一个综合案例，结合教程中多个知识点]

## 常见陷阱汇总
[生产环境中可能遇到的问题和解决方案]
```

---

## 工作流

1. **解析教程** — 提取所有知识点和它们的讲解深度
2. **缺口分析** — 用 7 个维度逐项检查每个知识点
3. **补全内容** — 对每个缺口生成深化内容
4. **整合输出** — 保留原教程结构，插入深化内容
5. **更新索引** — 调用 docs-manager 脚本更新索引：
   ```bash
   python .pi/skills/docs-manager/scripts/update-index.py deep-dives
   ```

---

## 护栏规则

1. **不重复原教程** — 只输出增量内容，不照搬原文
2. **标注引用** — 深化内容基于推理还是外部资料，需要注明
3. **代码可运行** — 新增代码示例必须注明运行环境
4. **不过度深化** — 如果原教程对某知识点已足够深入，跳过
