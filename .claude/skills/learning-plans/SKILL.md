# Learning Plans — 创建结构化学习计划

> 根据用户指定的领域，输出一套完整的学习计划：学习路径图、按知识点拆分的教程文件、
> 每节含概念讲解+代码示例+练习+扩展阅读、进度追踪文件。

---

## 核心定位

用户说"我想学 X"，本 Skill 负责输出一个零基础可执行的学习方案。
教程按知识点拆分，每个知识点一个独立 .md 文件，不限制总字数但控制单文件长度。

---

## 触发条件

- 用户说"我想学 [技术/领域]"
- 用户说"帮我制定 [领域] 的学习计划"
- 用户说"系统学习一下 [领域]"
- 任何需要从零开始掌握某个技术领域的请求

---

## 输出结构

在 `docs/learning-plans/active/<plan-slug>/` 下创建：

```text
<plan-slug>/
├── plan.md               ← 学习路径总览（目标、里程碑、时间估算）
├── progress.md           ← 进度追踪（每节打勾）
├── tutorials/
│   ├── INDEX.md          ← 教程目录索引
│   ├── 01-xxx.md         ← 第一节（概念 + 代码 + 练习 + 扩展）
│   ├── 02-xxx.md         ← 第二节
│   └── ...               ← 按知识依赖关系排序
└── resources.md          ← 推荐学习资源汇总
```

---

## 教程文件格式规范

使用 `templates/tutorial-template.md` 作为每个教程文件的标准模板。该模板定义了完整的区块结构（概念讲解 → 代码示例 → 练习 → 扩展阅读 → 常见陷阱），确保教程输出一致。

---

## 学习路径设计原则

1. **从具体到抽象** — 先给能跑起来的代码，再讲原理
2. **每个知识点独立可学** — 但明确标注前置依赖
3. **练习驱动** — 每节必有练习，学完即练
4. **螺旋上升** — 核心概念在多节中反复出现，逐步加深
5. **可跳过标注** — 进阶内容标注"可选"，让学习者自行把控深度

---

## 工作流

1. **需求澄清** — 确认领域范围、学习者当前水平、学习目标
2. **搭建骨架** — 运行脚手架脚本创建目录结构：
   ```bash
   python .pi/skills/learning-plans/scripts/scaffold-plan.py "<计划名称>"
   ```
3. **路径设计** — 参考 `docs/learning-plan-design.md` 中的拆分原则和覆盖清单，分解知识点，排序，估算每节耗时
4. **填充 plan.md** — 按 `templates/plan-template.md` 格式完善学习路径总览
5. **生成教程** — 按 `templates/tutorial-template.md` 逐节写出完整教程
6. **生成 resources.md** — 推荐阅读汇总
7. **标记进度** — 初始化 progress.md，所有节标记为未完成
8. **更新索引** — 调用 docs-manager 脚本更新索引：
   ```bash
   python .pi/skills/docs-manager/scripts/update-index.py learning-plans
   ```

---

## 护栏规则

1. **禁止空洞内容** — 每节必须有可运行的代码示例，不能只有理论
2. **知识点粒度** — 每个知识点可在一小时内学完（读+练）
3. **零占位符** — 禁止 TBD/TODO，每节必须是完整内容
4. **实际可运行** — 代码示例必须注明运行环境要求，确保可复现
