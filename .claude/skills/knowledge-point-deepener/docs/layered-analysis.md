# 七层剖析法详解

## 为什么是七层

大多数教程停在第二层（API 层）。七层框架确保从初学者到专家的每个
阶段都能获得增量价值。

## 各层标准

### 第 1 层: 直觉理解
- 必须用日常类比（不是技术类比）
- 一句话能跟非程序员说清楚
- 例: "Promise 就像一个餐厅取餐震动器——你点餐后不用在柜台等，
  拿到震动器继续做别的事，餐好了它会通知你。"

### 第 2 层: 使用场景
- 列举 3 个典型场景
- 列举 2 个不适用场景（反面教材）
- 包含决策树: if X then use Y else use Z

### 第 3 层: API 层
- 完整的参数表格（参数名/类型/默认值/说明）
- 返回值类型和含义
- 每个 API 至少一个使用示例

### 第 4 层: 行为契约
- 前置条件（调用前必须满足什么）
- 后置条件（调用后保证什么）
- 不变量（什么始终不变）
- 异常列表（什么情况抛什么异常）
- 线程安全声明

### 第 5 层: 实现原理
- 核心算法或数据结构的伪代码
- 不需要完整的实现，但要表达关键思想
- 包含复杂度分析

### 第 6 层: 源码分析
- 引用真实代码（标注文件路径和行号）
- 标注代码版本/commit hash
- 只引用关键片段，不粘贴整个文件
- 解释代码中的设计选择

### 第 7 层: 对比与边界
- 至少对比 2 个同类方案
- 对比维度: 性能、易用性、生态、学习曲线、适用规模
- 解释为什么选择了当前实现而非替代方案
- 讨论极端情况下的行为

## 深度等级决策表

| 知识点类型 | 建议深度 | 说明 |
|-----------|---------|------|
| 基础语法/API | 第 3 层 | 不需要源码分析 |
| 常用设计模式 | 第 5 层 | 到实现原理即可 |
| 语言核心机制 | 第 6 层 | 如 Rust 所有权、Go goroutine |
| 框架核心机制 | 第 6 层 | 如 React Fiber、Vue 响应式 |
| 分布式系统组件 | 第 7 层 | 需要对比方案 |
| 算法/数据结构 | 第 7 层 | 需要完整分析 |
