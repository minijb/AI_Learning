# 学习计划: C++ 游戏工程师详细攻略

> 创建日期: 2026-05-31
> 预计总耗时: 约 120 小时（30 节 × 平均 4 小时/节）
> 目标水平: 精通 — 能胜任 AAA 级游戏引擎核心开发

---

## 学习目标

完成本计划后，你将能够：

- 从编译到运行时，完整理解 C++ 在游戏引擎中的每一条执行路径
- 手写自定义分配器（池/栈/Arena/PMR），在每帧 16.6ms 预算内完成所有内存操作
- 运用模板元编程实现零开销抽象：CRTP、Concepts、编译期计算替代运行时开销
- 深刻理解 C++ 内存模型与原子操作，手写 Lock-Free 数据结构支持多线程 Job System
- 掌握数据导向设计 (DOD)，写出缓存友好的高性能 C++ 代码
- 运用 C++17/20/23 现代特性提升引擎代码的表达力与安全性，同时保持零开销承诺
- 看懂 Unreal Engine、Godot、Bevy 等开源引擎的 C++ 核心代码，并能贡献 Patch

## 前置要求

- [x] C++ 基础语法（类、继承、虚函数、模板基本用法）
- [x] 基础数据结构和算法
- [ ] 推荐：了解至少一个图形 API（OpenGL/Vulkan/DirectX）的基本概念
- [ ] 推荐：了解操作系统基础（进程/线程/虚拟内存）

> 如果你还没掌握 C++ 基础语法，建议先学习 `game-engine-dev` 计划中的第 0 节作为前置。

## 里程碑

- [ ] **阶段 0-1（第 1-4 节）**：掌握 C++ 编译流程、对象模型与 RAII 核心。里程碑：能正确设计引擎 RAII 资源类，理解编译器何时生成特殊成员函数。
- [ ] **阶段 2（第 5-7 节）**：掌握 C++ 值语义与所有权。里程碑：能写出零额外拷贝的资源管理代码，正确运用 unique_ptr 管理 GPU 资源。
- [ ] **阶段 3（第 8-12 节）**：掌握自定义内存管理。里程碑：能实现并集成自定义分配器（池/栈/Arena）到引擎中，理解引擎每帧内存的来龙去脉。
- [ ] **阶段 4（第 13-17 节）**：掌握编译期抽象。里程碑：能使用 CRTP、Concepts、constexpr 等编写零开销的引擎抽象层，让编译器代替运行时工作。
- [ ] **阶段 5（第 18-19 节）**：写出缓存友好的代码，初识 SIMD。里程碑：能将热路径数据结构从 AoS 改写为 SoA，性能提升 2-5 倍。
- [ ] **阶段 6（第 20-22 节）**：掌握并发编程。里程碑：能正确使用内存序实现 Lock-Free 队列，设计引擎 Job System 的核心数据结构。
- [ ] **阶段 7（第 23-26 节）**：掌握现代 C++ 新特性。里程碑：能用 Concepts 约束模板、用 Coroutines 写异步加载、用 expected 处理错误。
- [ ] **阶段 8（第 27-30 节）**：掌握引擎实战模式。里程碑：能设计零分配的引擎容器，实现属性反射系统，搭建性能剖视线。

## 使用建议

1. **顺序学习**：知识点按拓扑排序，强烈建议按顺序学习，不要跳课
2. **动手优先**：每节代码全部手敲一遍，不要复制粘贴
3. **笔记辅助**：已有 `docs/deep-dives/` 中多份 C++ 深度探索笔记可作为扩展阅读
4. **交叉参考**：本计划与 `game-engine-dev` 计划互补——本计划纯 C++，那个计划是引擎系统架构
