# 学习计划：C 系语言互操作与编译（C# ↔ C++ ↔ Lua）

> 创建日期: 2026-06-18
> 预计总耗时: 约 17 小时（13 节 × 60–90 分钟）
> 目标水平: 进阶 — 能在 C#、C++、Lua 三种语言之间设计并实现高效、正确的互操作层，并理解每条调用链的编译与运行机制

---

## 学习目标

完成本计划后，你将能够：

- 解释托管语言（C#）、原生语言（C/C++）、脚本语言（Lua）三种编译模型的差异，以及为什么 **C ABI 是跨语言通信的事实标准**。
- 用 `dumpbin` / `nm` 检查符号导出，理解 `extern "C"`、调用约定、名称重整如何影响互操作。
- 用 **P/Invoke（DllImport）** 和 **LibraryImport 源生成器** 让 C# 调用 C/C++ 库，并正确封送结构体、数组、字符串。
- 实现**反向调用**：让 C++ 通过函数指针 / `[UnmanagedCallersOnly]` 调用 C# 代码，并安全管理委托生命周期。
- 判断何时该用 **C++/CLI 包装器**、何时该用 **P/Invoke**、何时该用 **COM**。
- 用 **Lua C API** 写一个 C 扩展模块，理解 `lua_State` 栈模型。
- 用 **LuaBridge** / **sol2** 把 C++ 类绑定到 Lua；用 **LuaJIT FFI** 零绑定调用 C。
- 区分 **NLua** / **MoonSharp** / **xLua** / **toLua** 的原理与适用场景，理解 Lua ↔ C# 的 GC 桥与热更新（IL 注入）机制。
- 从零搭建一个 **C++ 核心 + C# 宿主 + Lua 脚本**的三层互通项目，并剖析端到端调用链。

## 前置要求

- [ ] C# 基础语法（类、方法、`unsafe`、`struct`）
- [ ] C++ 基础（类、指针、头文件、编译链接概念）
- [ ] Lua 基本语法（变量、表、函数）
- [ ] 命令行操作（`cd`、运行编译器）
- [ ] 建议已装：.NET 8 SDK、一个 C++ 编译器（MSVC 或 GCC/Clang）、CMake、Lua 5.4

## 里程碑

- [ ] 第一阶段完成（第 1–3 节）：能用工具看符号导出，解释为什么 C ABI 是通用边界，说清托管堆/原生堆与 GC pinning。
- [ ] 第二阶段完成（第 4–7 节）：能构建 C++ 动态库并从 C# 调用（P/Invoke + LibraryImport），实现双向调用，并选型 C++/CLI。
- [ ] 第三阶段完成（第 8–10 节）：能写 Lua C 扩展、用 LuaBridge/sol2 绑定 C++ 类、用 LuaJIT FFI 零绑定调 C。
- [ ] 第四阶段完成（第 11–12 节）：能解释 NLua/MoonSharp/xLua 的原理差异，说清 Lua ↔ C# 的 GC 桥与 xLua Hotfix 的 IL 注入机制。
- [ ] 最终项目（第 13 节）：搭建一个 C++ 数学核心（被 C# 通过 P/Invoke 调用）+ C# 宿主（内嵌 Lua 脚本，脚本可回调 C#）+ Lua 热修的完整端到端示例，并画出三语言调用链时序图。
