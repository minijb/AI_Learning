# 学习计划: 游戏引擎开发工程师

> 创建日期: 2026-05-19
> 预计总耗时: 234+ 小时
> 目标水平: 从零构建具备完整功能的迷你 3D 游戏引擎

---

## 学习目标

完成本计划后，你将能够：

- 理解现代游戏引擎的完整架构，从数学基础到系统层
- 手写核心子系统：渲染器、物理引擎、动画系统、音频引擎、资源管理器
- 实现 ECS 架构以支持高效的游戏逻辑执行
- 掌握多线程 Job System 以充分利用现代 CPU 多核能力
- 从零用 C++ 构建一个可运行的迷你 3D 游戏引擎，包含 3D 渲染、物理模拟、骨骼动画和音频播放
- 具备阅读 Unreal Engine / Godot / Bevy 等开源引擎源码的能力

## 前置要求

- [ ] **C++ 语言特性精要**（本章提供）：RAII、移动语义、智能指针策略、placement new、模板元编程、内存序 —— 以游戏引擎视角深入
- [x] 基础的数据结构与算法（树、图、哈希表、排序）
- [x] 操作系统基础（进程/线程、内存布局、虚拟内存）
- [ ] 推荐：了解一门图形 API 的基础（OpenGL / Vulkan / DirectX / Metal）

## 里程碑

- [ ] **预备阶段 (章节 0)**：掌握引擎开发必需的 C++ 语言特性。里程碑：能正确运用 RAII + 移动语义封装引擎资源类，理解内存序在并发中的应用。
- [ ] **第一阶段 (章节 1-5)**：掌握数学基础与引擎底层架构。里程碑：理解 ECS 架构原理，搭建跨平台窗口框架，掌握内存分配与 SIMD 优化基础。
- [ ] **第二阶段 (章节 6-9)**：掌握图形渲染核心。里程碑：能独立编写一个渲染 3D 模型并带有光照的场景。
- [ ] **第三阶段 (章节 10-13)**：掌握引擎核心子系统。里程碑：搭建包含资源管理、输入、动画、音频的完整子系统。
- [ ] **第四阶段 (章节 14-15)**：掌握物理模拟。里程碑：实现一个物理沙盒，多个刚体在重力下碰撞、弹跳。
- [ ] **第五阶段 (章节 16-18)**：掌握性能优化与高级渲染。里程碑：实现基于 PBR 的延迟渲染管线，理解多线程渲染架构和 Render Graph。
- [ ] **第六阶段 (章节 19-21)**：掌握工具链开发与网络同步。里程碑：搭建具备场景编辑、材质编辑功能的编辑器界面，了解网络同步原理。
- [ ] **第六阶段补充 (章节 22)**：掌握脚本系统集成。里程碑：实现 Lua/C# 脚本绑定、热重载机制、脚本与 C++ 的交互。
- [ ] **最终项目 (章节 23)**：从零构建迷你 3D 游戏引擎。里程碑：一个完整可执行的引擎，能加载模型、播放动画、模拟物理、输出音频，渲染一个可交互的 3D 场景。
- [ ] **职业阶段 (章节 24)**：了解开源引擎架构和职业发展路径。
