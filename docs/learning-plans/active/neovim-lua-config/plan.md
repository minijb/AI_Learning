# 学习计划: Neovim + Lua 配置实战 (现代版)

> 创建日期: 2026-06-02
> 预计总耗时: 约 16 小时
> 目标水平: 进阶 — 能独立用 Lua 配置和定制 Neovim，理解 kickstart.nvim 配置范式
> Neovim 版本: **0.12+**（本计划基于 0.12 的 `vim.pack`、`vim.lsp.config` 等新 API）

---

## 学习目标

完成本计划后，你将能够：

- 用 Lua 编写结构清晰的 Neovim 配置文件（**单文件 (kickstart) 与模块化两种风格都掌握**）
- 使用 **`vim.pack`**（Neovim 0.12 内置插件管理器）安装、更新、管理插件
- 独立配置 **原生 LSP**（`vim.lsp.config` / `vim.lsp.enable`）、**blink.cmp**（补全引擎）、Treesitter
- 自定义按键映射、自动命令、用户命令
- 深入理解 **kickstart.nvim** 的单文件架构设计，并能将其拆分为模块化结构
- 遇到配置问题时能定位到 Neovim 的对应 `:help` 文档

## 前置要求

- [x] 会使用 Neovim 的基本编辑操作（hjkl、模式切换、保存退出）
- [x] 有任意一门编程语言的基础（变量、函数、控制流）
- [ ] **Neovim 0.12+ 已安装**（本计划依赖 `vim.pack`、`vim.lsp.config` 等 0.12 新增 API）

## 里程碑

- [ ] **阶段一（第 1-3 节）：Lua 够用** — 能读懂和编写 Neovim 中常见的 Lua 配置片段
- [ ] **阶段二（第 4-6 节）：Neovim 配置基础** — 能用 init.lua 管理选项、按键映射和自动命令
- [ ] **阶段三（第 7-8 节）：现代插件管理** — 能用 `vim.pack` 安装和配置插件，理解 lockfile 机制
- [ ] **阶段四（第 9-11 节）：开发环境核心** — LSP + blink.cmp + Treesitter 三件套跑通
- [ ] **阶段五（第 12-13 节）：UI 与导航** — Telescope 模糊查找 + 外观定制
- [ ] **深入理解单文件架构（第 14 节）** — 逐行读懂 kickstart.nvim 的 init.lua，每个设计决策
- [ ] **掌握模块化架构（第 15 节）** — 将单文件拆分为标准模块化结构，理解拆分原则
- [ ] **最终项目（第 16 节）** — 搭建两套配置（单文件 + 模块化），可用作日常开发环境

## 本次更新说明（2026-06-02）

相对于旧版计划，以下内容已全面更新以反映 Neovim 0.12+ 生态：

| 旧（已过时） | 新（当前） |
|-------------|-----------|
| lazy.nvim 插件管理 | `vim.pack` 内置插件管理器 |
| `nvim-lspconfig` + `lspconfig.setup()` | `vim.lsp.config()` + `vim.lsp.enable()` (原生 API) |
| nvim-cmp 补全引擎 | blink.cmp 补全引擎 |
| 仅覆盖单文件配置 | **单文件 (kickstart) + 模块化多文件 两种风格全覆盖** |
| 无 lockfile 概念 | `nvim-pack-lock.json` 版本锁定 |
| 无模块化拆分教程 | **新增第 15 节：模块化配置架构与迁移策略** |
