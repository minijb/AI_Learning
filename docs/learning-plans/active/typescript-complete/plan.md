# 学习计划: TypeScript 完整学习

> 创建日期: 2026-06-13
> 预计总耗时: 16-20 小时（含最终项目）

## 学习目标

完成本计划后，你能够：

- 从零搭建 Node.js + TypeScript 工具开发环境，配置 `tsconfig.json` 和编译流程。
- 利用类型系统提前捕获错误，写出可维护、可重构的代码。
- 使用接口、类、泛型、高级类型工具对复杂领域进行建模。
- 处理联合类型、可选值、异步流程和错误边界，写出健壮的通用代码。
- 阅读并编写 `.d.ts` 声明文件，为无类型的 npm 包补齐类型。
- 使用 ESM / CommonJS 模块组织项目，理解模块解析规则。
- 完成一个可运行的命令行工具，综合运用所学知识。

> [!note] 学习背景假设
> 你已具备 C# / Lua 编程经验，对 JavaScript 有基础了解。本计划会借用 C# 的类型直觉，同时补齐 JS 运行时的关键概念（原型、事件循环、`this`、异步模型），最终落脚到 TypeScript 通用工具开发。

## 与 C# / Lua 的迁移提示

| C# 概念 | TypeScript 对应 | 关键差异 |
|---------|----------------|----------|
| `class` / `interface` | `class` / `interface` | TS 的 `interface` 只存在于编译期；类的继承方式类似，但 `private` 仅在类型层面 |
| `struct` / 值类型 | 无原生值类型；`readonly` 对象可部分模拟 | 引用语义仍是 JS 行为 |
| `List<T>` / `Dictionary<TKey, TValue>` | `T[]` / `Record<K, V>` / `Map<K, V>` | 数组、对象、`Map` 都可使用，按场景选择 |
| `out` / `ref` | 无 | TS 函数返回多值通常用对象或元组 |
| `async` / `await` | `async` / `await` + `Promise<T>` | 语法相似，但底层基于事件循环 |
| 泛型约束 `where T : class` | `T extends ...` | 语法为 `extends`，约束在编译期生效 |

Lua 背景的学习者请特别注意：

- TypeScript 的类型检查是**编译时**行为，运行时不存在类型信息。
- 数组索引从 `0` 开始。
- `nil` 在 TS 中对应 `null` / `undefined`，但默认不严格相等，建议开启 `strictNullChecks`。
