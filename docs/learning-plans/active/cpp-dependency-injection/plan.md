# 学习计划: C++ 依赖注入完整学习

> 创建日期: 2026-06-23
> 预计总耗时: 18-20 小时（含最终项目）
> 目标水平: 进阶 —— 能在 C++ 游戏工程中独立设计可测试、低耦合的依赖注入架构

---

## 学习目标

完成本计划后，你能够：

- 用准确的语言讲清**依赖注入（DI）**解决什么问题——为什么"类 A 直接 `new` 它依赖的类 B"会造成紧耦合、难测试、难复用。
- 辨析 **IoC（控制反转）、DIP（依赖倒置原则）、DI（依赖注入）** 三者的关系，不再混为一谈。
- 说出依赖注入的三种形式（**构造器注入 / Setter 注入 / 接口注入**）的适用场景与取舍。
- 在 C++ 中用**抽象基类 + 纯虚函数**定义依赖接口，并正确编写虚析构函数。
- 用**引用、指针、智能指针**三种方式在 C++ 构造器中注入依赖，并讲清所有权归属与生命周期。
- 写出**组合根（Composition Root）**——在 `main()` 或游戏启动入口一次性装配整个对象图，业务代码里不再 `new` 依赖。
- 用**模板与静态多态（CRTP）**实现零开销的编译期 DI，理解它何时适合游戏热循环、何时不适合。
- 用 **`std::function` 与类型擦除**注入回调/事件，构建灵活的伤害事件系统。
- 读懂 C# 的 **`Microsoft.Extensions.DependencyInjection`** 容器（Singleton/Scoped/Transient 三种生命周期），并能与 C++ 手动装配对照。
- 用 **`boost::di`** 在 C++ 中实现容器式自动装配，知道它与手动装配的取舍。
- 用 **Mock/桩** 替换真实依赖（数据库、网络、渲染），写出不依赖完整游戏环境的**单元测试**。
- 识别并规避 DI **反模式**：Service Locator、环境上下文注入、过度抽象。
- 独立完成一个综合项目：一个**可测试的游戏战斗系统**（角色 + 敌人 + 伤害计算 + 事件总线 + 日志），全部依赖通过注入提供，核心逻辑有单元测试覆盖。

> [!note] 学习背景假设
> 你是一名**游戏开发工程师**，有 C++ 基础（会写类、继承、指针、STL），用过或听过 C#/Unity。你想系统掌握依赖注入，但更想知道"在 C++ 里到底怎么落地"。本计划全程围绕一个**游戏战斗系统**示例演进：从"角色直接 `new` 一把铁剑"的紧耦合代码，一步步重构到"可注入、可测试、可替换"的架构。C++ 是主线（第 4-12、14、16 节），C# 作为对照（第 3、11、13 节尤其集中）。

## 贯穿示例：游戏战斗系统

本计划全程围绕一个**逐步演进的战斗系统**展开，避免每节换不同例子导致认知负担。核心角色与依赖：

| 角色 | 说明 | 首次出现 |
|------|------|----------|
| `IWeapon` / `Sword` / `Bow` | 武器接口与实现（可替换的伤害来源） | 第 1、4 节 |
| `ILogger` / `ConsoleLogger` | 日志接口与实现（横切依赖） | 第 4 节 |
| `Hero` | 角色类，依赖武器 + 日志（被注入方） | 第 1 节 |
| `EventBus` / `std::function` 回调 | 伤害事件、命中回调 | 第 9 节 |
| DI 容器 / 组合根 | 装配整个对象图的入口 | 第 7、11、12 节 |
| `MockWeapon` | 测试用的假武器 | 第 14 节 |

> 随着章节推进，这个系统会从"一个角色 + 一把剑"长成"角色 + 敌人 + 伤害计算 + 事件总线 + 测试"的完整架构。每节只新增属于该节知识点的部分。

## 与既有知识的迁移提示

| 你已熟悉的（游戏开发） | 依赖注入中的对应 | 关键差异 |
|------------------------|------------------|----------|
| 角色身上挂脚本/组件 | 依赖 | 组件是被"持有"，DI 强调依赖从**外部传入**而非内部创建 |
| `FindObjectOfType` / 全局单例 | Service Locator | DI 反对运行时"找"依赖，要求编译期/装配期就确定 |
| 接口/抽象类 | 依赖契约 | C++ 无 `interface` 关键字，用纯虚函数 + 虚析构函数模拟 |
| 对象池 / `new` 管理生命周期 | 服务生命周期 | DI 容器用 Singleton/Scoped/Transient 表达生命周期 |
| 热循环性能优化 | 模板静态多态 | 编译期多态消除虚函数开销，适合每帧调用的代码 |
| 编辑器里拖拽赋值 | 组合根装配 | 组合根是"代码版的拖拽赋值"——集中在一处完成 |

## 学习节奏建议

- **每天 1-2 节**：每节 45-90 分钟，含阅读 + 动手敲代码。**一定要把示例跑起来**。
- **先跑后想**：第 4-7 节每节代码都能用 `g++ -std=c++17` 直接编译运行，建议建一个 `di-lab/` 文件夹练手。
- **遇到性能疑问先记下**：游戏开发最关心开销。模板静态多态（第 8 节）会专门回应"虚函数到底贵不贵"。
- **测试节留足时间**：第 14 节和最终项目（第 16 节）需要写测试，预留 uninterrupted 时间。
- **对照 C# 思考**：如果你写过 Unity/C#，每节末尾问自己"这在 C# 里会怎么写"，加深理解。
