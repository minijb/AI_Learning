# 学习计划: 帧同步、状态同步与状态帧同步

> 创建日期: 2026-06-02
> 预计总耗时: 约 38-44 小时（34 节 × 平均 65-75 分钟）
> 目标水平: 精通（资深游戏工程师面试级别）

---

## 学习目标

完成本计划后，你将能够：

1. **从零设计**一套帧同步(Lockstep)系统，理解确定性逻辑、定点数数学库、帧对齐、乐观帧锁定等核心技术
2. **实现完整的**状态同步(State Sync)系统：客户端预测、服务端和解、实体插值、延迟补偿
3. **设计混合同步**架构（状态帧同步），懂得何时用帧同步、何时用状态同步，实现双通道混合方案
4. **掌握 Unity / Unreal / Lua** 三个引擎下的同步方案实现细节和最佳实践
5. **应对面试**中的网络同步深度问题：浮动点数确定性、反外挂策略、断线重连、录像回放、观战系统
6. **理解 AAA 级项目**（王者荣耀帧同步、守望先锋状态帧同步、合金弹头混合同步）的架构决策

---

## 前置要求

- [ ] C#、C++ 或 Lua 至少一门语言熟练（教程三版本对照，理解一种即可）
- [ ] 基础网络知识（TCP/UDP 区别、Socket 编程的基本概念）
- [ ] 游戏引擎基础（Unity 或 Unreal 的基本使用）
- [ ] 线性代数和基础物理（用于碰撞检测和位置同步）

## 学习方法建议

1. **按顺序学习**：每节都有明确的前置依赖，跳过会导致概念断层
2. **三版本对照**：Unity/Unreal/Lua 三版本实现建议完整阅读对照，理解"原理→平台适配"的思维过程
3. **写代码**：每节至少完成一个练习。光读不写 = 没学
4. **手写关键算法**：定点数数学库、Dead Reckoning、Server Rewind 建议手写一遍
5. **研究开源项目**：resources.md 中列出的开源项目是重要学习资源

## 面试重点

以下知识点是面试高频问点，请重点掌握：
- 帧同步的确定性如何保证（定点数、随机种子、浮点陷阱）
- 乐观帧锁定的实现原理和优缺点
- 客户端预测 + 服务端和解 + 实体插值的三件套
- 服务端回退(Server Rewind)的命中判定逻辑
- 状态帧同步混合同步的架构决策依据
- 断线重连的不同实现策略对比
