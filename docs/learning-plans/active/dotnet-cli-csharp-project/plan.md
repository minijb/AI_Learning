# 学习计划: dotnet CLI 与 C# 工程构建

> 创建日期: 2026-06-10
> 预计总耗时: ~12 小时（每节约 60 分钟，含阅读 + 代码练习）
> 目标水平: 入门 — 能熟练使用 dotnet CLI 创建、构建、测试、发布 C# 工程

---

## 学习目标

完成本计划后，你能：

1. 使用 `dotnet` 命令行工具创建、构建、运行、清理 C# 项目
2. 理解 `.csproj` 和 `.sln` 文件结构，手动编辑项目配置
3. 管理解决方案中的多项目关系（项目引用、NuGet 包引用）
4. 使用 `dotnet test` 编写和运行单元测试
5. 使用 `dotnet publish` 和 `dotnet pack` 发布应用和类库
6. 配置多目标框架、条件编译、自定义 MSBuild 属性
7. 使用 `dotnet tool` 和 `dotnet ef` 等扩展命令
8. 从零搭建一个完整的多层 C# 解决方案

## 前置要求

- [x] 基本的命令行操作（`cd`、`ls`、管道等）
- [x] C# 基础语法（类、方法、命名空间）— 如果完全没有，建议先学 C# 语法基础
- [x] .NET SDK 已安装（建议 .NET 8+），可通过 `dotnet --version` 验证

## 学习路径

### 第一阶段：环境与基础命令（3 节）

| 序号 | 知识点 | 预计耗时 | 类型 | 前置 |
|------|--------|---------|------|------|
| 1 | dotnet 环境安装与验证 | 40min | 基础 | 无 |
| 2 | `dotnet new` — 创建项目与解决方案 | 60min | 核心 | 1 |
| 3 | `dotnet build` / `run` / `clean` — 编译运行与清理 | 60min | 核心 | 2 |

### 第二阶段：依赖与解决方案管理（3 节）

| 序号 | 知识点 | 预计耗时 | 类型 | 前置 |
|------|--------|---------|------|------|
| 4 | `dotnet add` / `remove` — 管理包引用与项目引用 | 60min | 核心 | 3 |
| 5 | `dotnet sln` — 解决方案管理 | 50min | 核心 | 4 |
| 6 | 项目文件结构深入 — `.csproj` 与 `.sln` | 60min | 核心 | 5 |

### 第三阶段：测试、发布与打包（3 节）

|------|--------|---------|------|------|

### 第四阶段：扩展工具与实战（3 节）

| 序号 | 知识点 | 预计耗时 | 类型 | 前置 |
