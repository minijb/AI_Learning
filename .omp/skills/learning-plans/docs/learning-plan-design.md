# 学习计划设计指南

## 知识点拆分原则

### 粒度标准
- 每个知识点应可在 30-60 分钟内完成（阅读 + 练习）
- 如果一个概念需要超过 1 小时，拆分它
- 如果一个概念不到 15 分钟，考虑合并到相邻知识点

### 排序原则
1. 按知识依赖关系拓扑排序 — 先学依赖项
2. 难度由浅入深 — 先给简单可运行的代码建立信心
3. 核心概念优先 — 高频使用的知识点排在前面
4. 穿插练习 — 每 2-3 节安排一个综合练习节

### 命名规范
- 文件名: `NN-slug.md`（NN 是序号，slug 是英文短名）
- 例: `01-variables-and-types.md`、`02-control-flow.md`

## 领域覆盖清单

设计学习路径时，检查是否覆盖：
- [ ] 核心概念和原理
- [ ] 环境搭建和工具链
- [ ] 基础语法/API
- [ ] 常用模式和最佳实践
- [ ] 常见陷阱和反模式
- [ ] 测试和调试
- [ ] 性能考量和优化思路
- [ ] 与其他技术的对比和选型建议
- [ ] 实战项目
