# 教程深化策略

## 判断教程深度的标准

### 浅层教程的特征
- 只有代码没有解释
- 只有 API 用法没有原理
- 只讲 happy path 不讲错误处理
- 没有性能讨论
- 没有对比其他方案
- 例: "用 fetch 发请求: fetch(url).then(res => res.json())"

### 深层教程的特征
- 从"为什么"出发
- 包含底层原理（内存布局、协议细节）
- 讨论边界情况和错误处理
- 有性能数据和 benchmark
- 有与其他方案的对比

## 深化优先级

按以下顺序补充（最紧缺的先补）：
1. **前置知识** — 没有它读者根本看不懂后面
2. **原理推导** — 理解了原理才能举一反三
3. **常见陷阱** — 避免读者踩坑是最直接的 ROI
4. **边界情况** — 面试和生产的常见考点
5. **对比方案** — 帮助建立技术选型框架
6. **性能特征** — 进阶内容，可在实际需要时再讲
7. **演进历史** — 锦上添花，不必须

## 深化程度控制

- 每个知识点的深化不超过原教程该节长度的 50%（避免信息过载）
- 如果某个缺口需要长篇解释，建议拆分为独立深度探索
- 标记深度等级: 🔵 基础补充 / 🟢 进阶延伸 / 🔴 专家级深入
