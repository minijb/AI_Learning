# 通用 Markdown 书写规范

此 skill 覆盖标准 CommonMark 加 GFM 扩展，适用于 GitHub、GitLab、VS Code、文档平台等绝大多数 markdown 渲染器。目标是写出**语义清晰、渲染稳定、可维护**的 markdown。

## 核心语法

### 标题

```markdown
# H1 — 页面唯一标题
## H2 — 主要章节
### H3 — 子章节
#### H4 — 少用，层级不宜超过 H4
```

标题级别反映内容层次，不要为了视觉效果跳级（如 H1 直接跳 H3）。每个文档通常只有一个 H1。

### 段落与换行

```markdown
这是第一段。

这是第二段。段落之间用一个空行分隔。

行尾加两个空格  
可以强制换行，但尽量少用——换一段更清晰。
```

### 强调

```markdown
*斜体* 或 _斜体_
**粗体** 或 __粗体__
***粗斜体***
~~删除线~~
```

### 列表

无序列表统一用 `-`，不混用 `*` 和 `+`：

```markdown
- 第一项
- 第二项
  - 嵌套项（缩进 2 空格）
  - 嵌套项
- 第三项
```

有序列表编号从 `1.` 开始，后续编号不影响渲染，但保持语义上连续：

```markdown
1. 第一步
2. 第二步
3. 第三步
```

列表项内容较长时，用空行分隔会让每项渲染为段落（松散列表）——仅在必要时使用：

```markdown
- 这是一个短项

- 这是一个需要更多说明的项。

  缩进 2 空格后可以追加段落或代码块。
```

### 代码

行内代码用单反引号：

```markdown
使用 `git commit -m "msg"` 提交更改。
```

代码块用三反引号，**务必标注语言**以启用语法高亮：

````markdown
```python
def hello():
    print("Hello, World!")
```
````

#### 嵌套代码块

当代码块内容中还有代码块时，外层反引号数必须严格多于内层。规则：**外层反引号数 > 内容中最长连续反引号数**。

````markdown
```markdown
这是内层代码块示例
```
````

如果内层用了三个反引号，外层改用四个：

`````markdown
````markdown
```bash
echo "hello"
```
````
`````

不确定时，宁可多加一个反引号。

### 链接与图片

```markdown
[链接文本](https://example.com)
[带标题的链接](https://example.com "悬停提示")
[引用式链接][ref-id]

[ref-id]: https://example.com

![图片替代文本](./images/screenshot.png)
![带标题的图片](./images/diagram.png "图表说明")
```

图片 `alt` 文本要有描述意义，不要留空或写 `image`。

### 表格（GFM）

表格前后各留一个空行：

```markdown
| 列 A   | 列 B   | 列 C   |
|--------|:------:|-------:|
| 左对齐 | 居中   | 右对齐 |
| 值 1   | 值 2   | 值 3   |
```

对齐语法：`|---|` 左对齐，`|:---:|` 居中，`|---:|` 右对齐。

### 引用块

```markdown
> 这是一段引用。
>
> 引用可以跨段落。

> [!NOTE]
> GitHub 支持的提示块（Alerts）。

> [!WARNING]
> 类型：NOTE、TIP、IMPORTANT、WARNING、CAUTION
```

### 任务列表（GFM）

```markdown
- [x] 已完成
- [ ] 待完成
- [ ] 另一项
```

### 水平分割线

```markdown
---
```

三个 `-`，前后各留空行，避免与 frontmatter 或标题混淆。

### 转义

特殊字符前加 `\` 转义：

```markdown
\*不是斜体\*
\[不是链接\]
\`不是代码\`
```

常见需转义字符：`\ * _ [] () # + - . !`

### Frontmatter（如平台支持）

部分平台（GitHub Pages、Hugo、Jekyll）支持 YAML frontmatter，必须是文件第一行：

```markdown
---
title: 文档标题
date: 2026-05-19
tags: [tag1, tag2]
---
```

## 书写规则

1. **标题唯一性**：每个文档只有一个 H1，作为文档标题
2. **标题不跳级**：层级连续，H2 之后接 H3，不要直接跳 H4
3. **列表符号统一**：同一文档内无序列表只用 `-`
4. **代码块标注语言**：所有代码块必须写语言标识符，无明确语言时用 `text`
5. **嵌套代码块加反引号**：内容中含 ` ``` ` 时外层用 ```` ` ``` ```` ，以此类推
6. **表格前后空行**：表格与上下文之间各留一个空行
7. **图片 alt 文本有意义**：描述图片内容，不要留空
8. **链接文本有意义**：用描述性文字，不要用"点击这里"或裸 URL
9. **尖括号包反引号**：正文中的泛型、HTML 标签、尖括号表达式用反引号包裹，如 `List<T>`，防止被解析为 HTML
10. **段落间空行**：段落、标题、列表、代码块、表格之间保留空行，确保解析器正确分块
11. **避免行尾空格**：除非需要强制换行，行尾不留空格

## 常见错误

```markdown
❌ # 标题
内容紧跟标题没有空行          → 部分解析器不识别为标题

✅ # 标题

内容与标题之间有空行
```

```markdown
❌ ```                         → 没有语言标识，无语法高亮
✅ ```python
```

```markdown
❌ [点击这里](https://...)     → 链接文本无意义
✅ [查看安装文档](https://...)
```
